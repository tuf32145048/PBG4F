# コンテンツ拡張ワークフロー

Python初学者向け教材を継続的に増やすための標準手順です。毎回この流れで、章単位の追加、検証、記録を揃えます。

## 基本方針

- 1回の標準作業は「1章 + 8から12問 + 必要な用語 + 必要なTips」にする。
- 章の目的は、学習者が次に解ける問題の種類で定義する。
- 本文は短く、問題で練習量を確保する。
- 新しい概念は本文、問題、用語の3か所で役割を分ける。
- 用語は表示語、説明対象、カテゴリ、自動リンク可否を先に決める。
- 一般語や頻出語は本文中でリンクだらけにならないよう、原則として関連用語側で扱う。
- 長期判断が必要な設計思想は、その場で決めず `notes/questions/` に質問化する。

## 固定ループ

1. `git status --short --branch` を確認する。
2. `notes/TODO.md` と `notes/content-roadmap.md` で次の章候補を確認する。
3. `content-addition-template.md` のLesson Briefを埋める。
4. 章のゴール、前提、扱わない内容を決める。
5. `term-plan-template.md` を使い、`notes/term-plans/YYYY-MM-DD-<lesson-slug>.md` に用語候補をまとめる。
6. 用語の表示語、説明対象、カテゴリ、自動リンク可否についてユーザー確認を受ける。
7. Lesson JSONとMarkdownを作る。
8. 問題JSONをLevel 1からLevel 3の順に作る。
9. 用語とTipsを追加・調整する。
10. `verification-checklist.md` で確認する。
11. `npm run validate` を実行する。
12. 必要に応じて `npm run test`、`npm run lint`、`npm run build` を実行する。
13. `notes/TODO.md` に作業記録、検証結果、残課題を追記する。

## ファイル編集順

標準の編集順は次の通りです。

1. `notes/TODO.md`
2. `notes/content-roadmap.md`
3. `notes/term-plans/YYYY-MM-DD-<lesson-slug>.md`
4. `web/content/lessons/<slug>.json`
5. `web/content/lessons/<slug>.md`
6. `web/content/problems/<slug>.json`
7. `web/content/terms/terms.json`
8. `web/content/tips/tips.json`
9. 必要なテストまたは実装ファイル

コンテンツだけで完結する作業では、`web/src/` を変更しません。表示要件が既存UIで表現できない場合だけ、UI変更として別タスクに分けます。

## 章設計の基準

各章は次の問いに答えられる状態で書き始めます。

- この章の前に、学習者は何を知っているか。
- この章の後に、学習者はどんな問題を解けるか。
- この章で初めて出す概念は何か。
- この章では、あえて扱わない概念は何か。
- 問題は簡単な確認から統合問題へ自然に進むか。

答えが曖昧なまま本文を書き始めると、章が肥大化しやすくなります。

## 問題セットの標準配分

- Level 1: 3から4問。単一概念の確認。
- Level 2: 4から6問。2つ以上の処理を順に組み合わせる。
- Level 3: 1から2問。問題文を分解して実装する。

各問題には、段階ヒント、解答、思考トレース、典型ミス、見直し項目を必ず入れます。問題数を増やすより、1問ごとの復習可能性を優先します。

## 当面の不足領域

現在の3章では、表示、値、変数、演算、入力、型変換までを扱っています。Beginnerガイドとして拡張するには、少なくとも次の領域が必要です。

- 比較演算と真偽値
- `if` / `elif` / `else` による条件分岐
- `for` と `range` による反復
- `while` と終了条件
- リスト、添字、要素の更新
- 文字列操作
- 合計、最大、最小、個数の集計
- 関数の基本
- 辞書の基本
- エラーの切り分けと小さなデバッグ

この順番は固定ではありませんが、初学者の負荷を考えると、まず条件分岐、反復、リストへ進めるのが自然です。

## 質問ファイルを作る基準

次の判断が必要な場合は、実装を止めずに `notes/questions/YYYY-MM-DD-topic.md` へ質問を作ります。

- 教本としての説明順に複数の妥当案がある。
- 初学者へどこまで厳密に説明するか迷う。
- 問題の題材や語調が教材方針に関わる。
- Pyodideや実行環境など、大きな依存やUI変更に波及する。

質問ファイルには、背景、質問、選択肢、推奨案、回答欄を書きます。

## 完了条件

章追加は次を満たしたら完了扱いにします。

- Lesson JSONとMarkdownがある。
- 対応する問題JSONがある。
- 参照する用語とTipsが存在する。
- `npm run validate` が成功している。
- 必要な場合は `test`、`lint`、`build` が成功している。
- `notes/TODO.md` に記録が残っている。
